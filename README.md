### Утилита обновления VR‑шлемов Meta Quest

[![Screenshot](https://habrastorage.org/r/w1560/getpro/habr/upload_files/d90/601/c97/d90601c9779d8e1232911929ba44604a.png "Screenshot")](https://habr.com/ru/articles/847242/)

Небольшая desktop‑утилита с графическим интерфейсом на Python/Tkinter для обновления прошивки шлемов **Meta Quest** (Quest, Quest 2, Quest 3, Quest 3S, Quest Pro) через ADB.

> Подробнее о мотивации и разработке можно почитать в статье на Хабре:  
> [Утилита обновления VR‑шлема Meta Quest](https://habr.com/ru/articles/847242/)

---

### Возможности

- **Автоопределение устройства по ADB**
  - Проверка подключения шлема.
  - Определение модели (Quest / Quest 2 / Quest 3 / Quest 3S / Quest Pro) по `ro.product.model`.
  - Отображение статуса подключения и подсказок в интерфейсе.

- **Работа с прошивками**
  - Получение списка актуальных прошивок с официальных страниц Meta (см. `firmware_archive.py`).
  - Фильтрация и сортировка версий.
  - Выбор нужной версии прошивки из выпадающего списка.
  - Возможность указать уже скачанный файл прошивки.

- **Загрузка прошивки**
  - Скачивание прошивки через `requests` с отображением прогресса.
  - Менеджер загрузок `DownloadManager` (`download_manager.py`) с поддержкой отмены и обработки ошибок.

- **Установка прошивки**
  - Запуск обновления прошивки на подключённом шлеме через ADB.
  - Подробный лог в текстовом поле внизу окна.
  - Пошаговая индикация: выбор ADB → подключение устройства → выбор/загрузка прошивки → обновление.

### Требования

- **ОС**: Windows 10/11.
- **Python**: 3.10+ (рекомендуется; см. `.python-version`).
- **ADB**: установленный `adb.exe` (Android Platform Tools).
- **Интернет**: нужен для автоматического скачивания прошивок.

Зависимости Python описаны в `pyproject.toml` (см. секцию `project.dependencies`).

Рекомендуемый способ — [uv](https://github.com/astral-sh/uv) с использованием `sync`:

```bash
# установка uv (если ещё не установлен)
pip install uv

# установка зависимостей согласно pyproject.toml
uv sync
```

После этого можно запускать приложение так:

```bash
uv run main.py
```

> `uv sync` сам создаст и привяжет виртуальное окружение, отдельные команды `python -m venv` / `pip install` не нужны.

Опционально, классический вариант через `pip`:

```bash
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

### Установка и запуск

1. Клонируйте репозиторий:

```bash
git clone https://github.com/<USER>/<REPO>.git
cd <REPO>
```

2. Установите зависимости (рекомендуется `uv sync`, см. раздел «Требования»).

3. Запустите приложение:

```bash
uv run main.py
```

4. В окне программы:
   - Нажмите кнопку выбора `adb.exe` и укажите путь к ADB.
   - Нажмите «Проверить подключение по ADB» и убедитесь, что шлем обнаружен.
   - При необходимости скорректируйте модель шлема в комбобоксе.
   - Выберите версию прошивки из списка или скачайте последнюю доступную.
   - После подготовки прошивки запустите процесс обновления.

Все сообщения и возможные ошибки выводятся в текстовое поле лога.

### Структура проекта

- `main.py` — основной GUI на `Tkinter`, логика шагов обновления, взаимодействие с ADB.
- `firmware_archive.py` — получение и сортировка ссылок на прошивки Meta Quest.
- `download_manager.py` — менеджер фоновых загрузок (прогресс, отмена, обработка ошибок).

---

### Предупреждение

- Любые операции с прошивкой выполняются **на ваш страх и риск**.
- Перед обновлением рекомендуется:
  - Зарядить шлем минимум до 50–60%.
  - Не отключать кабель во время прошивки.
  - По возможности сделать резервную копию важных данных.
